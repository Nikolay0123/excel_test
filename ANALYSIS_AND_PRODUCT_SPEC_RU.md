# Анализ Excel и целевое Android-приложение

## 1) Что делает исходная таблица

Файл `Кафе Джем учет, кухня (нов) (1).xlsx` ведет помесячный учет питания:

- каждый лист = отдельный месяц (`Ноябрь 26`, `Декабрь 26`, `Январь 26`, ...);
- строки гостей/организаций содержат:
  - номер комнаты/идентификатор,
  - имя или организацию,
  - тип оплаты (`Да` / `Нет` / `Налич`);
- по дням месяца хранятся отметки количества питания по 3 типам:
  - `З` (завтрак),
  - `О` (обед),
  - `У` (ужин);
- справа идут автоматические итоги по каждой строке:
  - количество завтраков/обедов/ужинов,
  - сумма по строке,
  - разрез по способу оплаты.

Внизу листа:

- сводные суммы по месяцу;
- цены на завтрак/обед/ужин (без НДС и с НДС);
- финансовый блок (налоги, издержки, прибыль, остатки по наличным/безналу).

## 2) Цели и задачи этого учета

Основные задачи бизнеса:

- фиксировать факт питания по каждому гостю/организации в разрезе дней;
- разделять оплату по трем схемам: с НДС, без НДС, наличные;
- быстро получать:
  - количество приемов пищи,
  - выручку/суммы по месяцу,
  - предварительную прибыль и остатки.

## 3) Что реализовано в Android-версии

Создан новый проект `CafeJemAccounting`:

- **без табличного ввода**: данные вносятся через формы и кнопки;
- **разделы интерфейса**:
  - `Гости` — заведение карточек гостей/организаций;
  - `Ввод` — добавление записей питания (день, тип питания, порции);
  - `Итоги` — агрегаты и экспорт;
- **хранение данных локально**: Room (`Guest`, `MealEntry`, `MealRate`);
- **экспорт**: кнопка `Экспорт в XLSX` формирует файл Excel.

## 4) Архитектура

- UI: Jetpack Compose;
- State: `AppViewModel`;
- Data: Room + Repository;
- Export: Apache POI (`XSSFWorkbook`).

## 5) Что уже сделано на втором этапе

Добавлено:

- экран `Финансы`:
  - редактирование ставок завтрака/обеда/ужина;
  - редактирование параметров налогов (НДС %, налог %);
  - добавление ежемесячных издержек с каналом `cash/bank`;
- расчет финансовых показателей:
  - выручка (всего/с НДС/без НДС/наличные),
  - расходы (всего/наличные/безнал),
  - НДС, налог, прибыль,
  - остатки наличных и на счете;
- расширенный экспорт:
  - основной лист с данными по гостям,
  - отдельный финансовый лист с ключевыми метриками и списком издержек.

## 6) Что осталось для полного parity 1:1

- экспорт в строго идентичный макет исходного Excel (матрица `день * З/О/У` как в исходнике);
- формулы финансового блока, полностью повторяющие логику книги построчно;
- импорт существующих данных/справочников из старой книги.
